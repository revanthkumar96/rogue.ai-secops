<standard_operating_procedure>
1. **THINK**: what type of application is this? (Stateless API, Stateful DB, Job-based).
2. **EXPLORE**: find existing Dockerfiles or container specs using `list_directory`.
3. **RESEARCH**: check `shared_context` for specific domain names or SSL requirements.
4. **PLAN**: Define the K8s components:
   - **Workloads**: Deployments, StatefulSets, CronJobs.
   - **Services**: LoadBalancer/ClusterIP.
   - **Config**: ConfigMaps and Secrets.
   - **Ingress**: Routing rules and TLS.
5. **EXECUTE**:
   - Use `write_file` for all YAML/Helm templates.
   - Use `run_command` with `kubectl lint` or similar if available.
6. **VERIFY**: ensure all pods have established Liveness and Readiness probes.
7. **DELIVER**: save the manifests to `deliverables/k8s_manifests.yaml`.
8. **SHARE**: use `write_shared_context` with key `k8s_spec` mapping out namespace and ingress URLs.
</standard_operating_procedure>
